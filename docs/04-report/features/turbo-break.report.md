## 7. 성공 요인 (Keep)

### 7.1 설계의 완성도

✅ **명확한 스펙** — Plan v2.1에서 10가지 조건의 경계값/목적/이론적 배경을 정확히 정의. 덕분에 구현 중 "뭘 만들지?" 같은 의문 없음.

✅ **체계적 업그레이드** — v1(기본) → v2(신기능) → v2.1(개선) 3단계로 진행. 각 단계마다 설계 → 구현 → 분석. 이 피드백 루프가 99% Match Rate 달성.

✅ **계층별 책임 분리** — Domain(screener.ts) / Infrastructure(adapters) / Presentation(UI) 명확. 덕분에 logger.ts 추가해도 다른 파일 영향 없음.

### 7.2 기술 선택

✅ **어댑터 패턴** — Mock/Yahoo/Kiwoom 3개를 환경변수 1줄로 전환. 유연성과 확장성 극대화.

✅ **TypeScript + Zod** — 외부 데이터 검증으로 런타임 안정성 확보.

✅ **Tailwind v4 다크모드** — CLAUDE.md 패턴 완벽 준수로 night mode 자연스러움.

### 7.3 개발 프로세스

✅ **Gap 분석 반복** — Analysis v1→v2→v3→v4 4번 반복. 매번 발견된 이슈가 설계/구현에 반영. "설계-구현-검증" 선순환.

✅ **관찰 가능성** — 모든 구현이 설계와 1:1 대응 가능. "이 기능은 Design Section 7.4입니다" 같은 추적 가능.
