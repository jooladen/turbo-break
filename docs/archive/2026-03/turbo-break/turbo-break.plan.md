## 1. 배경 및 목적

### 1.1 문제 정의

- 매일 장 마감 후 수백~수천 종목 중에서 직접 차트를 확인하는 것은 비효율적
- 테마주·급등주는 단기 변동성이 커 다음날 진입 시 손실 리스크가 높음
- 박스권 횡보 후 수급을 동반한 "안전한 돌파" 종목을 자동으로 필터링하는 도구가 필요
- 주식 초보자가 결과를 이해하기 어렵고, 전문가 수준의 분석 해설이 없어 활용이 제한됨

### 1.2 목표

- 10가지 정량적 조건으로 박스권 돌파 종목을 자동 스크리닝
- 테마주·급등주를 제외하고 추세 기반의 안전한 돌파 종목만 선별
- **BuySignal 점수/등급 시스템**으로 종목 품질을 직관적으로 표현
- 왕초보부터 전문가까지 수준별 분석 해설 제공
- 매일 장 마감 후 즉시 사용 가능한 웹 기반 스크리너 제공

## 3. 스크리닝 조건 (10가지)

| # | 조건명 | 기준 | 목적 |
|---|--------|------|------|
| 1 | 20일 고가 돌파 | 종가 > 20일 최고가 | 박스권 상단 돌파 확인 |
| 2 | 횡보 필터 | (20일 최고가 − 20일 최저가) / 20일 최저가 ≤ 15% | 테마주·급등주 제외 |
| 3 | 거래량 폭증 | 당일 거래량 ≥ 20일 평균 × 2 | 수급 동반 확인 |
| 4 | 윗꼬리 제거 | 종가 ≥ 당일 고가 × 0.99 | 힘 있는 돌파만 선별 |
| 5 | 거래대금 | 거래대금 ≥ 500억 원 | 유동성 충분한 종목만 |
| 6 | 추세 확인 | 종가 > 60일 이동평균 | 단기 반등 아닌 추세 돌파 |
| 7 | 과열 방지 | 당일 상승률 < 8% | 다음날 안전 진입 가능 |
| 8 | 양봉 | 종가 > 시가 | 차익 실현 세력 약한 종목 |
| 9 | 갭 제한 | 시가 ≤ 전일 종가 × 1.03 | 갭 돌파 제외 (갭 메움 리스크) |
| 10 | 연속 상승 제한 | 직전 5일 누적 상승률 ≤ 15% | 이미 많이 올라온 종목 제외 |

## 4. BuySignal 시스템 (v2 신규)

### 4.1 매수 점수 체계

각 조건에 가중치를 부여하여 0~100점으로 환산:

| 조건 | 가중치 | 근거 |
|------|--------|------|
| 20일 고가 돌파 | 10점 | 핵심 진입 시그널 |
| 횡보 필터 | 9점 | VCP 패턴 확인 |
| 거래량 폭증 | 9점 | 수급 신뢰도 |
| MA60 위 | 8점 | 중기 추세 확인 |
| 거래대금 | 8점 | 유동성 확보 |
| 5일 급등X | 7점 | 추격매수 방지 |
| 갭X | 7점 | 진입 리스크 관리 |
| 과열X | 6점 | 단기 과열 방지 |
| 양봉 | 6점 | 당일 매수세 확인 |
| 윗꼬리 | 5점 | 매도 압력 부재 확인 |

### 4.2 매수 등급 기준

| 등급 | 점수 범위 | 의미 |
|------|-----------|------|
| A | 80점 이상 | 강력 추천 — 지금 살 수 있어요! |
| B | 65~79점 | 괜찮아요 — 살 만한 종목이에요 |
| C | 45~64점 | 좀 더 기다려요 — 아직은 애매해요 |
| F | 44점 이하 | 사지 마세요 — 지금은 위험해요 |

### 4.3 분석 탭 구조

| 탭 | 대상 | 내용 |
|----|------|------|
| 📊 차트 탭 | 모든 사용자 | lightweight-charts 캔들차트 (MA5/20/60, 20일 고가선, 돌파 마커) |
| 🐣 왕초보 분석 탭 | 주식 초보자 | 등급/점수/긍정이유/걱정점을 어린이 언어로 설명 |

## 7. 아키텍처 개요

```
외부 주가 API (Yahoo Finance / 키움 API)
    ↓
lib/adapters/           ← Yahoo, 키움 어댑터
lib/market-data.ts      ← 어댑터 인터페이스 + MockAdapter + createAdapter(type?)
    ↓
lib/screener.ts         ← 10가지 조건 순수 함수 + BuySignal 평가
    ↓
app/api/screener/route.ts   ← GET /api/screener?market=ALL&adapter=mock
    ↓
app/(dashboard)/screener/
    ├── page.tsx                ← Server Component (force-dynamic)
    ├── ScreenerControls.tsx    ← 시장/날짜/어댑터 선택
    └── ScreenerTable.tsx       ← Client Component (전체 UI)
components/
    ├── stock-chart-interactive.tsx  ← 캔들차트 (lightweight-charts)
    └── theme-toggle.tsx             ← 다크모드 토글
```

**설계 원칙**:
- 계산 로직을 순수 함수로 분리 → 테스트 가능
- 데이터 소스를 어댑터로 추상화 → 런타임 전환 가능
- Server Component 기본 → 클라이언트 번들 최소화

## 8. 데이터 소스 전략

| 어댑터 | 환경변수 값 | 설명 |
|--------|------------|------|
| `MockAdapter` | `mock` (기본값) | 시드 기반 난수, KOSPI 20 + KOSDAQ 15종목 |
