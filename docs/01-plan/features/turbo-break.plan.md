## 1. 배경 및 목적

### 1.1 문제 정의

- 매일 장 마감 후 수백~수천 종목 중에서 직접 차트를 확인하는 것은 비효율적
- 테마주·급등주는 단기 변동성이 커 다음날 진입 시 손실 리스크가 높음
- 박스권 횡보 후 수급을 동반한 "안전한 돌파" 종목을 자동으로 필터링하는 도구가 필요

### 1.2 목표

- 10가지 정량적 조건으로 박스권 돌파 종목을 자동 스크리닝
- 테마주·급등주를 제외하고 추세 기반의 안전한 돌파 종목만 선별
- 매일 장 마감 후 즉시 사용 가능한 웹 기반 스크리너 제공

## 3. 스크리닝 조건 (10가지)

| # | 조건명 | 기준 | 목적 |
|---|--------|------|------|
| 1 | 20일 고가 돌파 | 종가 > 20일 최고가 | 박스권 상단 돌파 확인 |
| 2 | 횡보 필터 | (20일 최고가 − 20일 최저가) / 20일 최저가 ≤ 15% | 테마주·급등주 제외 |
| 3 | 거래량 폭증 | 당일 거래량 ≥ 20일 평균 × 2 | 수급 동반 확인 |
| 4 | 윗꼬리 제거 | 종가 ≥ 당일 고가 × 0.99 | 힘 있는 돌파만 선별 |
| 5 | 거래대금 | 거래대금 ≥ 500억 원 | 유동성 충분한 종목만 |
| 6 | 추세 확인 | 종가 > 60일 이동평균 | 단기 반등 아닌 추세 돌파 |
| 7 | 과열 방지 | 당일 상승률 < 8% | 다음날 안전 진입 가능 |
| 8 | 양봉 | 종가 > 시가 | 차익 실현 세력 약한 종목 |
| 9 | 갭 제한 | 시가 ≤ 전일 종가 × 1.03 | 갭 돌파 제외 (갭 메움 리스크) |
| 10 | 연속 상승 제한 | 직전 5일 누적 상승률 ≤ 15% | 이미 많이 올라온 종목 제외 |

## 6. 아키텍처 개요

```
외부 주가 API (v2)
    ↓
lib/market-data.ts   ← 어댑터 인터페이스 + MockAdapter (v1)
    ↓
lib/screener.ts      ← 10가지 조건 순수 함수
    ↓
app/api/screener/route.ts   ← GET /api/screener
    ↓
app/(dashboard)/screener/   ← UI (Server + Client Component)
```

**설계 원칙**:
- 계산 로직을 순수 함수로 분리 → 테스트 가능
- 데이터 소스를 어댑터로 추상화 → KIS API 등 교체 용이
- Server Component 기본 → 클라이언트 번들 최소화

## 7. 데이터 소스 전략

| 단계 | 구현체 | 설명 |
|------|--------|------|
| v1 (현재) | `MockAdapter` | 시드 기반 난수로 KOSPI 20종목 + KOSDAQ 15종목 생성 |
| v2 | `KisAdapter` | 한국투자증권 Open API 연동 |
| v2 대안 | `YfinanceAdapter` | Python 서버 경유 yfinance 데이터 |

어댑터 교체 시 `lib/market-data.ts`의 `createAdapter()` 함수만 수정하면 됨.
