## 1. 배경 및 목적

### 1.1 문제 정의

- 기존 일봉 스크리너는 조건이 타이트해 **실제 장에서 걸리는 종목이 없는 날**이 있음
- 일봉 돌파 종목을 발견해도 "언제 사야 하는가"에 대한 **분봉 기반 진입 타이밍**이 없음
- 초보자는 종목을 발견해도 당일 어느 가격에 매수해야 할지 판단하기 어려움

### 1.2 기회

- **ORB(Opening Range Breakout)** 는 학술 논문으로 검증된 인트라데이 전략
- 분봉 기반이므로 **매일 반드시 시그널 발생** (일봉 스크리너의 단점 보완)
- 기존 키움 REST API 어댑터를 분봉 API로 **자연스럽게 확장** 가능

### 1.3 목표

| # | 목표 | 측정 기준 |
|---|------|-----------|
| 1 | 일봉 스크리너와 인트라데이 뷰를 연결하는 워크플로우 제공 | "📊 ORB" 버튼 클릭 → `/intraday` 이동 |
| 2 | ORB 고가/저가 + VWAP 시각화 | 차트에 점선 + 라인 표시 |
| 3 | 돌파 시그널 자동 감지 및 요약 표시 | LONG/SHORT 카드 렌더링 |
| 4 | Mock 데이터로 개발 환경에서 즉시 동작 | `pnpm dev` 후 `/intraday` 접속 |

## 2. 사용자 스토리

```
일봉 스크리너에서 관심 종목 발견
    ↓ "📊 ORB" 버튼 클릭
/intraday?ticker=005930 접속
    ↓
분봉 차트 확인 (09:00~15:30)
    ↓
ORB 박스(오프닝 레인지 고가/저가) 확인
    ↓
VWAP 라인과 현재가 위치 비교
    ↓
"🟢 LONG 09:47 진입" 시그널 카드 확인
    ↓
실제 증권사 HTS에서 매수 실행
```

## 4. 범위 (Scope)

### In Scope (v1.0)

- ORB 고가/저가 계산 (5분/15분/30분 선택)
- VWAP 누적 계산 (09:00부터)
- 돌파 시그널 감지 (거래량 1.5배 필터)
- 분봉 캔들차트 시각화 (lightweight-charts)
- Mock 합성 데이터 (KRX 09:00~15:30, GBM 기반)
- 일봉 스크리너 → 인트라데이 연결 (ORB 버튼)

### Out of Scope (v1.0)

- 실시간 분봉 데이터 연동 (키움 WebSocket)
- 복수 종목 동시 ORB 모니터링
- 알림 (푸시/이메일)
- 백테스팅 기능
